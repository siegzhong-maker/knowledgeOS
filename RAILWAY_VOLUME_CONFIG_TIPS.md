# Railway Volume 配置提示词

## 配置 Volume 时的填写说明

### Volume Mount Path（挂载路径）
```
/data/uploads
```

**说明**：
- 这是容器内的路径，PDF文件将存储在这里
- 即使容器重新部署，Volume中的数据会保留
- 与代码中的配置保持一致

### Volume 名称（如果有名称字段）
```
uploads
```

**说明**：
- 可选的标识名称，方便管理
- 可以是任意名称，不影响功能

## 完整的配置参数

如果Railway要求填写多个字段，参考以下：

| 字段 | 值 | 说明 |
|------|-----|------|
| **Mount Path** | `/data/uploads` | 容器内挂载路径（必需） |
| **Name** | `uploads` | Volume名称（可选） |
| **Size** | 默认或最小 | Volume大小（可选，根据需要调整） |

## 配置后的验证

配置完成后，检查：

1. **部署日志中应该看到**：
   ```
   ✓ 上传目录已准备: /data/uploads
   ```

2. **验证文件持久化**：
   - 上传一个测试PDF文件
   - 触发重新部署
   - 检查PDF文件是否仍然可访问

## 如果遇到问题

### 路径相关
- 确保路径以 `/` 开头（绝对路径）
- 不要使用相对路径或 `~` 符号

### 权限相关
- Railway会自动处理权限
- 如果遇到权限错误，联系Railway支持

### 存储限制
- 免费计划可能有存储大小限制
- 检查Railway文档了解限制
